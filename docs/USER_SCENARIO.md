# 마이링크 (MyLink) 사용자 시나리오

해당 문서는 마이링크 서비스의 전체적인 핵심 경험(Core Loop)을 중심으로 방문자와 소유자 측면에서 발생할 수 있는 가능한 모든 시나리오를 정의합니다. 통계 항목에 대해서는 '링크 클릭 수'만을 집계하여 프로필을 개선해 나가는 순환 구조를 따릅니다.

---

## 1. 코어 루프 (Core Loop): 마이링크 생성부터 통계 확인까지

### 단계 1: 서비스 온보딩 및 프로필 생성 (소유자)
- 사용자는 **복잡한 절차 없이 빠르게 회원 가입을 진행**하기 위해 **구글 계정을 통한 소셜 로그인 버튼을 클릭**한다.
- 사용자는 **본인만의 식별 가능한 접속 주소를 만들기** 위해 **가입 직후 부여될 고유 닉네임(URL)과 프로필 최상단에 보여질 한 줄 소개를 입력**한다.
- 사용자는 **다른 사람과 주소가 겹치거나 시스템 오류가 발생하는 것을 방지**하기 위해 **시스템 예약어나 이미 존재하는 닉네임을 입력했을 때 나타나는 경고 메시지를 확인하고 다른 닉네임을 다시 입력**한다.

### 단계 2: 링크 추가 및 관리 (소유자)
- 사용자는 **자신의 포트폴리오, SNS, 블로그 등을 방문자가 한눈에 볼 수 있도록 모아두기** 위해 **대시보드 화면에서 '새 링크 추가' 버튼을 눌러 각 타겟 URL과 제목을 반복적으로 등록**한다.
- 사용자는 **오타가 있거나 정보가 바뀐 링크를 수정**하기 위해 **인라인 편집(Inline Edit) 기능을 통해 변경할 텍스트 영역을 클릭하고 즉시 텍스트를 수정하여 저장**한다.
- 사용자는 **더 이상 노출할 필요가 없는 링크를 방문자에게 보이지 않게 정리**하기 위해 **해당 링크 항목 우측의 '삭제' 버튼을 눌러 목록에서 완전히 제거**한다.

### 단계 3: 마이링크 공유 (소유자)
- 사용자는 **외부 사람들을 자신의 통합 프로필로 효과적으로 유도**하기 위해 **완성된 본인의 마이링크 고유 주소(`mylink.com/nickname`)를 복사한 뒤 인스타그램 바이오(Bio)나 이력서 등에 게시**한다.

### 단계 4: 페이지 유입 및 링크 클릭 (방문자)
- 사용자는 **특정 크리에이터의 최신 활동이나 전체 채널 목록을 확인**하기 위해 **크리에이터가 공유한 마이링크 URL로 접속**한다.
- 사용자는 **자신이 들어온 페이지의 주인이 본인이 찾던 사람이 맞는지 식별**하기 위해 **화면 최상단에 자동으로 연동된 구글 프로필 이미지와 닉네임, 한 줄 소개를 확인**한다.
- 사용자는 **등록된 여러 링크 중 목적에 맞는 사이트를 시각적으로 쉽게 구별**하기 위해 **링크 버튼에 자동으로 추출되어 표시된 각 사이트의 파비콘 이미지를 확인**한다.
- 사용자는 **관심 있는 특정 콘텐츠(예: 최신 유튜브 영상, 기술 블로그 등)를람하기** 위해 **목록에서 원하는 내용이 적힌 직관적인 링크 버튼을 클릭하여 해당 페이지로 이동**한다.

### 단계 5: 통계 확인 및 프로필 최적화 (소유자)
- 사용자는 **자신이 등록한 여러 채널 중 어떤 콘텐츠가 방문자에게 가장 인기가 많은지 파악**하기 위해 **대시보드에서 각 개별 링크별로 누적된 '클릭 수(Click Count)' 통계를 확인**한다.
- 사용자는 **통계 데이터를 바탕으로 프로필의 활용도를 극대화**하기 위해 **클릭이 저조한 링크는 삭제하고, 더 인기 끌 만한 새로운 목적지의 링크를 지속적으로 추가(등록)**한다. *(→ 단계 2로 다시 순환하며 코어 루프 완성)*

## 2. 🚀 개선점 제안 (추후 고도화 고려 사항)

현재 1차 기획(PRD) 스펙에 맞춰 코어 루프 시나리오가 작성되었으나, 소유자의 지속적인 리텐션(재방문율) 및 사용성을 더욱 높이기 위해 다음 기능을 제안합니다.

1. **링크 숨김(비활성화) 토글 기능**
   - **이유**: 특정 기간에만 진행하는 짧은 이벤트 링크나 잠시 내려두고 싶은 포트폴리오 링크의 경우, 매번 누적된 클릭 수 통계까지 같이 '삭제'해 버리는 것은 데이터 손실 관점에서 아쉽습니다. 삭제 대신 시야에서만 가리고 통계는 유지할 수 있는 '숨김/노출 토글' 스위치가 있으면 사용자 편의가 매우 커집니다.
   - **시나리오**: "사용자는 **이벤트가 끝난 링크의 누적 클릭 통계는 보존하되 방문자에게는 잠시 보여주지 않기** 위해 **해당 링크의 노출 토글 버튼을 '비활성' 상태로 변경**한다."

2. **드래그 앤 드롭을 통한 링크 순서 변경 기능**
   - **이유**: 사용자는 통계(단계 5)를 살피며 "클릭률이 높은 핵심 링크를 맨 위로 올려야겠다"는 인사이트를 얻게 마련입니다. 현재는 단순 생성일(createdAt) 순서로만 보여 매번 링크를 다 지웠다 다시 써서 순서를 맞추는 불편함이 생길 수 있습니다.
   - **시나리오**: "사용자는 **가장 트래픽을 집중시키고 싶은 핵심 링크가 방문자 눈에 가장 먼저 띄게 하기** 위해 **해당 링크 블록을 마우스로 드래그하여 목록 최상단으로 끌어올린다.**"
