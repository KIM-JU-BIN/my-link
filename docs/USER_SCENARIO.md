# 마이링크 (MyLink) 사용자 시나리오

해당 문서는 마이링크 서비스의 경험을 **방문자(Visitor)**와 **사용자/소유자(Owner)** 두 그룹으로 명확히 나누어 발생할 수 있는 가능한 모든 시나리오를 정의합니다. 작성된 시나리오는 서비스의 핵심 코어 루프(가입-생성-공유-유입-통계)를 포괄하며, 오직 통계는 '클릭 수 카운트' 기능만을 포함하는 것을 전제로 합니다.

---

## 1. 방문자 시나리오 (Visitor)

*방문자는 소유자가 외부 채널에 공유한 링크를 통해 유입되어 제공된 콘텐츠를 소비하는 역할을 합니다.*

- 사용자는 **특정 크리에이터의 최신 활동이나 전체 채널 목록을 한번에 통합적으로 확인**하기 위해 **크리에이터가 공유한 마이링크 고유 URL(`mylink.com/nickname`)로 접속**한다.
- 사용자는 **자신이 들어온 페이지의 주인이 본인이 찾던 사람이 맞는지 식별**하기 위해 **화면 최상단에 자동으로 연동되어 노출된 구글 프로필 이미지와 닉네임, 한 줄 소개를 확인**한다.
- 사용자는 **소유자가 등록한 여러 링크 중 본인의 목적에 맞는 사이트를 시각적으로 쉽게 구별**하기 위해 **각 링크 버튼에 자동으로 추출되어 표시된 해당 사이트의 파비콘 이미지를 확인**한다.
- 사용자는 **관심 있는 특정 콘텐츠(예: 최신 유튜브 영상, 기술 블로그, 포트폴리오 등)를람하기** 위해 **목록에서 원하는 내용이 적힌 직관적인 링크 버튼을 클릭하여 해당 타겟 페이지로 이동**한다.

---

## 2. 사용자(소유자) 시나리오 (Owner)

*소유자는 서비스에 가입하여 본인의 다중 링크 프로필을 개설하고 링크를 등록 및 관리하며, 발생한 링크 클릭 통계를 확인하는 주체 역할을 합니다.*

### 가입 및 온보딩
- 사용자는 **복잡한 추가 절차 없이 빠르고 간편하게 프로필을 생성**하기 위해 **구글 계정을 통한 소셜 로그인 버튼을 클릭**한다.
- 사용자는 **본인만의 식별 가능한 고유 접속 주소(URL)를 만들기** 위해 **가입 직후 프로필 주소가 될 닉네임과 타인에게 보여질 한 줄 소개 텍스트를 입력**한다.
- 사용자는 **다른 유저와 주소가 겹치거나 라우팅 오류가 발생하는 것을 사전에 방지**하기 위해 **시스템 예약어나 이미 사용 중인 닉네임을 입력했을 때 나타나는 경고 메시지를 확인하고 다른 닉네임을 다시 입력**한다.

### 링크 등록 및 관리
- 사용자는 **자신의 흩어진 결과물(포트폴리오, SNS, 블로그 등)을 방문자가 한눈에 볼 수 있도록 모아두기** 위해 **대시보드 화면에서 '새 링크 추가' 버튼을 눌러 각 타겟 URL과 제목을 반복적으로 등록**한다.
- 사용자는 **입력 중 발생한 오타를 잡거나 정보가 변경된 링크를 지체 없이 바로잡기** 위해 **인라인 편집(Inline Edit) 기능을 통해 수정할 텍스트 영역을 클릭하고 즉시 새로운 텍스트를 입력 및 저장**한다.
- 사용자는 **더 이상 유효하지 않거나 노출할 필요가 없는 링크를 프로필에서 지우기** 위해 **해당 링크 항목 우측의 '삭제' 버튼을 눌러 목록에서 완전히 제거**한다.

### 외부 채널 공유
- 사용자는 **타 플랫폼의 지인이나 팬, 채용 담당자 등 외부 사람들을 자신의 통합 마이링크 프로필로 효과적으로 유도**하기 위해 **완성된 본인의 고유 주소(`mylink.com/nickname`)를 복사한 뒤 인스타그램 바이오(Bio)나 이력서 등에 게시**한다.

### 클릭 통계 확인 (점검 루프 완성)
- 사용자는 **자신이 등록한 여러 링크 중 어떤 콘텐츠가 방문자에게 가장 호응을 얻었는지 객관적인 수치로 파악**하기 위해 **대시보드에서 각 개별 링크별로 누적된 '클릭 수(Click Count)' 통계를 지속적으로 확인**한다.

---

## 3. 🚀 개선점 제안 (추후 고도화 고려 사항)

기존 기획을 바탕으로 작성되었으나 사용자 편의를 크게 높이기 위해 도입하면 좋을 기능들입니다.

1. **링크 숨김(비활성화) 토글 기능**
   - **이유**: 특정 기간에만 진행하는 짧은 이벤트 링크나, 당장 며칠만 내려두고 싶은 포트폴리오 링크의 경우 무조건 지워야 한다면 누적된 클릭 수 통계까지 같이 잃어버리게 됩니다. 시야에서만 가리고 통계 데이터와 정보는 유지할 수 있는 '숨김/노출 토글' 스위치가 필수적입니다.
   - **시나리오**: "사용자는 **이벤트가 끝난 링크의 누적 클릭 통계는 보관하되 방문자에게는 잠시 띄우지 않기** 위해 **해당 링크의 노출 토글 버튼을 '비활성' 상태로 조작**한다."

2. **드래그 앤 드롭을 통한 링크 순서 변경 기능**
   - **이유**: 통계를 살피다 보면 "클릭률이 높은 핵심 링크를 맨 위로 올려야겠다"는 아이디어를 얻기 쉽습니다. 현재는 무조건 생성일(createdAt) 순서로만 강제되어 매번 위로 올리고 싶은 링크를 전부 지웠다가 순서에 맞춰 다시 타이핑해야 하는 불필요한 공수가 생깁니다.
   - **시나리오**: "사용자는 **가장 방문자의 클릭 트래픽을 집중시키고 싶은 핵심 링크가 프로필에 접속한 이들의 눈에 1순위로 띄게 하기** 위해 **해당 링크 블록을 마우스로 드래그하여 목록 최상단으로 단번에 끌어올린다.**"
