# 마이링크 (MyLink) - PRD (제품 요구사항 정의서)

## 1. 프로젝트 개요
- **프로젝트 명**: 마이링크 (MyLink)
- **목적**: 여러 개의 링크를 하나의 통합된 프로필 페이지로 모아 쉽고 빠르게 공유할 수 있는 다중 링크 큐레이션 서비스 제공.
- **대상 사용자**: 
  - 다양한 채널(인스타그램, 유튜브, 깃허브, 블로그 등)을 운영하는 크리에이터 및 인플루언서
  - 포트폴리오, 이력서, 프로젝트 링크 등을 하나의 링크로 정리하고자 하는 구직자 및 개발자

---

## 2. 핵심 기능 목록

### 2.1. 필수 기능
1. **사용자 인증**
   - **단일 소셜 로그인**: Firebase Authentication을 활용한 **구글 소셜 로그인** 전용
   - **고유 프로필 주소**: `@` 기호 없이 사용자가 설정한 '닉네임'을 그대로 고유 URL로 할당 (예: `mylink.com/닉네임`)
   - **예약어 제한 로직**: 시스템 컴포넌트 라우팅(예: `login`, `admin`, `dashboard` 등)과 사용자 URL이 충돌하지 않도록 가입 단계에서 특정 키워드 닉네임 사용 금지
2. **프로필 관리**
   - **수정 가능 항목**: 닉네임, 한 줄 소개 (대시보드에서 텍스트 기반으로 언제든지 직접 수정 가능)
   - **구글 프로필 이미지 연동**: 직접 이미지를 업로드하는 기능은 존재하지 않으며, 구글 회원가입 시 연동되는 **Google 프로필 이미지**를 자동으로 렌더링
3. **링크 블록 관리**
   - **링크 목록 노출**: 사용자가 기존에 등록해둔 링크들을 단순 리스트 형태로 표시
   - **새 링크 추가 버튼**: 새로운 링크 URL과 제목을 등록
   - **링크 항목 조작 (수정/삭제)**: 
     - **파비콘 자동 연동**: 타겟 URL 입력 시 **구글 파비콘 API(Google Favicon API)**를 활용하여 아이콘 이미지를 자동으로 가져와 링크 버튼 디자인으로 설정
     - **인라인 편집(Inline Edit)**: 수정 모드 진입 시 모달창이나 페이지 이동 없이, 기존 링크 아이템 UI 내에서 즉시 텍스트(제목/URL)를 수정 및 저장 가능
4. **퍼블릭 페이지**
   - 접속 시 상단에 '구글 프로필 이미지', '닉네임', '한 줄 소개'가 보여지며, 그 아래로 파비콘이 포함된 깔끔한 링크 리스트 직관적 제공
   - **shadcn UI** 기반의 디자인 적용으로 모바일에 최적화된 반응형 뷰 렌더링

### 2.2. 추후 제공 예정
- **방문자 통계 및 링크 클릭 조회수**: 나중에 2차 고도화 기능으로, 개별 타겟 링크별 클릭수(조회수) 데이터를 트래킹하여 대시보드에 표기할 예정

### 2.3. 제외 기능
초기 버전에 집중하고 복잡한 관리 동선을 취소화하기 위해 다음 기능들은 1차 기획에서 일괄 배제합니다:
- 프로필 이미지 직접 업로드/수정 기능 제외 (구글 이미지로 대체)
- 생성된 링크 블록들을 사용자가 마우스로 드래그 앤 드롭하여 순서를 변경하는 기능 제외
- 특정 링크의 노출 상태를 활성화/비활성화시키는 토글 기능 제외
- 대시보드 한 켠에서 변화된 모습을 보여주는 실시간 모바일 뷰어 미리보기 화면 제외

## 3. 기능 상세 설명

### 3.1. 인증
- Firebase Authentication의 구글 로그인(Google Provider) 옵션만을 유일한 회원가입 및 접속 수단으로 구축합니다.
- 로그인 처리가 성공(Success)된 시점에 아직 고유 닉네임이 없는 최초 가입자의 경우, 추가 화면을 통해 '닉네임'과 '한 줄 소개'를 필수 입력받고 닉네임 중복 및 예약어 차단 여부를 검증하는 온보딩 절차를 밟습니다.

### 3.2. 대시보드
- 대시보드는 자신의 프로필(닉네임/한 줄 소개)과 링크 리스트를 한눈에 파악하고 관리하는 중앙 통제 화면입니다.
- 불필요하게 탭(Tab)이나 페이지를 여러 번 이동하는 것을 방지하기 위해 '새 링크 폼 열기', '기존 내용 텍스트 수정' 등의 행위가 모두 인라인 편집(Inline Edit) 형태로 해당 항목 내부에서 동적으로 전환되어 저장되도록 UI를 세팅합니다.

### 3.3. 디자인 시스템
- 대시보드 관리자 화면과 방문자가 보는 퍼블릭 공유 화면 모두 **shadcn UI** 컴포넌트를 베이스로 구축됩니다. 
- 복잡한 데스크탑 와이드 레이아웃 대신 모바일을 최우선 고려(Mobile-first)하여, 화면 가로폭이 넓어지더라도 디스플레이 중앙에 세로형 직사각형의 카드 형태 뷰어가 정렬되어 보이는 트렌디하고 미니멀한 구성 요소를 지향합니다.

## 4. 데이터베이스 아키텍처 (Firestore 기준)

NoSQL인 Firebase Firestore 구조를 채택하여, **User Collection**과 **Links Sub-collection**으로 역할을 명확히 나누어 모델링합니다.

### 📌 User Collection (`users`)
- `[document: uid]` (Firebase Auth UID 기준)
  - `id`: 유저 고유 식별자 (클라이언트 내부 조회를 위해 uid값 중복 저장)
  - `email`: 사용자 이메일 계정 (구글 소셜 로그인 연동 시 제공됨)
  - `displayName`: 표시 이름 (구글 소셜 로그인 연동 시 제공되는 이름)
  - `username`: 실제 사용자의 이름
  - `nickname`: 사용자 닉네임 겸 URL 식별자 (예: /nickname)
  - `bio`: 한 줄 소개 텍스트
  - `profileImageUrl`: 구글 소셜 로그인 연동 시 제공받는 기본 접속자 프로필 원본 사진 경로 저장
  - `createdAt`: 계정 첫 가입(생성) 일자

### 📌 Links Sub-collection (`users/{uid}/links`)
- `[document: linkId]` (Firestore 자동 생성 ID 기준)
  - `id`: 링크 고유 식별자 (클라이언트 내부 조회를 위해 linkId값 중복 저장)
  - `title`: 링크에 표시할 제목
  - `url`: 연결될 실제 타겟 URL
  - `faviconUrl`: 구글 파비콘 API를 통해 추출 및 저장된 이미지 URL
  - `createdAt`: 링크를 처음 추가한 일자 (단순 목록 등록일순 정렬의 1차 기준으로 활용)
